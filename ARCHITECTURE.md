# System Architecture (Living Document)

_Note: This is a living document. As the proof of concept evolves, this architecture will be updated to reflect new engineering decisions._

## 1. High-Level Design

The Medical Assistant Bot utilizes a decoupled, 3-tier architecture. It replaces standard semantic vector RAG with a "Reasoning-based RAG" approach using the PageIndex framework. This design mimics secure, event-driven pipelines, prioritizing data validation, asynchronous processing, and strict state management.

## 2. Tier Breakdown

### A. Presentation Tier (Frontend UI & Dashboards)

- **Technology:** Streamlit (Python)
- **Location:** `frontend/app.py`
- **Purpose:** \* Provides the interactive chat UI for doctors to submit clinical notes and query complex medical histories.
  - Houses hard-coded, visual statistical dashboards (using libraries like Pandas/Matplotlib) for medical students to track patient trends.
- **Network:** Communicates exclusively with the Logic Tier via HTTP REST requests. It holds no secrets and connects to no databases directly.

### B. Logic Tier (Backend Orchestrator & AI Engine)

- **Technology:** FastAPI, PageIndex Framework (Vectify AI), Google Gemini 1.5 Pro
- **Location:** `backend/` directory
- **Purpose:** \* **FastAPI (`main.py`):** Acts as the API gateway and traffic controller. Handles asynchronous routing, strict payload validation (via Pydantic), and database read/writes.
  - **PageIndex & Gemini (`index_engine.py`):** Executes the iterative reasoning loop. Instead of keyword vector searches, it drives Gemini to navigate a hierarchical JSON tree of the patient's history, evaluating context before extracting raw clinical notes to synthesize a final answer.

### C. Data Tier (Persistence & Indexing)

- **Technology:** Google Cloud Firestore (NoSQL)
- **Location:** Interfaced via `backend/database.py`
- **Purpose:** \* **Raw Clinical Notes:** Stores unedited, unstructured JSON documents submitted by doctors.
  - **In-Context Index:** Stores the dynamic JSON "Table of Contents" tree generated by the PageIndex framework to map the patient's timeline.

## 3. Deployment Strategy (Current: Local POC)

- **Frontend Server:** Running locally on `http://localhost:8501`.
- **Backend Server:** Running locally on `http://localhost:8000`.
- **Database:** Cloud-hosted (Google Firebase).
- **Compute Engine:** Cloud-hosted (Google Gemini API).
